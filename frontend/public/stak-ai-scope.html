How it works

AI Model

Gemini 2.5 Flash with no Google Search grounding. Live market data is injected from our own cached stock endpoint — keeps latency low and cost minimal.

Live Market Data

When a user mentions any brand or ticker, the backend self-calls /api/stock/:ticker for fresh price, change%, P/E, beta, and market cap — injected into the conversation before the AI responds.

Personalization

Every conversation is seeded with the user's Stak (with purchase prices), financial familiarity level, top interest tags, and the current ET date. The AI calibrates its explanation depth automatically.

What it knows about the user

Injected into every conversation as a system context turn before any user message is sent to Gemini.

Their Stak
All saved brands with ticker symbols and the price when they added it — e.g. "Nike (NKE) — added at $85.20". Gives the AI direct awareness of what they track and at what cost basis.
Financial familiarity
Beginner / intermediate / advanced — set during onboarding. The AI adjusts explanation depth and vocabulary to match.
Top interest tags
Top 3 weighted tags derived from swipe history (e.g. Tech, Fashion, Food & Beverage). Used to frame answers and generate personalized starter prompts.
Live stock data
If the user's message mentions any brand or ticker from the full 333-brand catalog (not just their Stak), we fetch live price data and inject it as a separate turn before their question. The AI is told to use those exact numbers.
Today's date (ET)
Injected so the AI reasons correctly about market sessions, earnings timing, and relative references like "this week" or "today" — without drifting into wrong timezone assumptions.
UI & entry point

A floating sparkles button sits fixed to the bottom-right of every in-app screen, above the bottom nav. Tapping opens a full-screen overlay with two distinct states.

Landing state · no messages yet
  • Large "Stak AI" heading and subtitle
  • Search-bar style input near the top
  • Four personalized "Try asking" rows with icons and chevrons
  • Tapping any prompt sends it immediately
  • History and new-chat controls in the top bar
Chat state · active conversation
  • Full message thread — user right, AI left with sparkle avatar
  • AI messages render bold text and bullet lists
  • Animated three-dot indicator while waiting
  • Textarea pinned to bottom, Enter to send
  • Optimistic append with automatic rollback on error

Guardrails & limits
✕

No buy / sell / hold recommendations. The system prompt explicitly prohibits directional advice. Any investment-adjacent response includes "for educational purposes only, not financial advice."

20

20 messages per user per day. Enforced in Postgres using a per-user ET day boundary. The 429 response surfaces as an in-app toast. Resets at midnight ET.

↺

Gemini refusal filter. Responses matching the known Gemini refusal pattern are caught and retried on the next available API key before surfacing an error — same shared filter used across all AI routes.

⇄

Key rotation with concurrency limit. Cycles through multiple Gemini API keys with 429-aware fallback. A shared concurrency limiter prevents request pile-ups under load.
